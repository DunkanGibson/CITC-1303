6.(a) You get city, state, and zip code information from the ZIPCODE table. 
  (b)  You can get student names from the STUDENT table.
  (c)  The relationship between ZIPCODE and STUDENT tables is 1:M. The ZIP field link them.
  (d)  SELECT stud.last_name, zip.city, zip.state, zip.zip
       FROM STUDENT STUD, ZIPCODE ZIP
       WHERE stud.zip = zip.zip
       ORDER BY zip.state


7.(a)  The STUDENT_ID in the ENROLLMENT table signifies a student’s enrollment in a class.
  (b)  The relationship between tables STUDENT and ENROLLMENT is 1:M. The STUDENT_ID field links them.
  (c)  You use the DISTINCT keyword remove duplication in the result of a query.
  (d)  SELECT DISTINCT stud.first_name, stud.last_name, e.student_id
       FROM STUDENT STUD, ENROLLMENT E
       WHERE stud.student_id = e.student_id
       ORDER BY e.student_id

8.(a) You can get the course description from the COURSE table.
  (b)  You can get the total capacity of a course from the SECTION table.
  (c)  The relationship between tables COURSE and SECTION is 1:M with the COURSE_NO field linking them?
  (d)  The SECTION_NO field in table SECTION can be used to find the number of sections of a course.
  (e)  SELECT c.description, SUM(s.capacity), MAX(s.section_no)
       FROM COURSE c JOIN SECTION s
       ON (c.COURSE_NO = s.COURSE_NO)
       GROUP BY c.description
       HAVING MAX(s.section_no) > 1
